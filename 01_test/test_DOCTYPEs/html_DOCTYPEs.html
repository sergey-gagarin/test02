Intro

Currently we have a choise of the most popular and widely supported markup languages HTML4.01 and XHTML 1.0 and recent enhancement - HTML 5. More information about MathML, Compound, Optional and old HTML2, HTML3 documents doctype declarations is available on W3C Quality Assuranced site.
At the same source we can read that HTML 5 is NOT a standard yet, so it can not pass the validation test. In case validation is vital for websites and the small icon to prove adhering to the standards is what you are after, HTML4.01 or XHTML 1.0 will do the job. But in case validation is not crucial and advance HTML 5 tags can not be replace, declear <!DOCTYPE html>. After all, Google use it.
Not a bad idea to check the list of attributes supported in different DOCTYPEs on on w3school site .
Essential that in HTML4.01 and XHTML 1.0 Transitional versions the following elements are not supported: <frame />, <frameset>
In HTML4.01 and XHTML 1.0 Strict versions the following elements are not supported: <frame />, <frameset>, <applet>, <basefont />, <center>, <dir>, <font>, <iframe>, <isindex>, <menu>, <noframes>, <s>, <strike>, <u> <br />
The Frameset version supports all of the above elements.

XHTML

According to W3C, XHTML is not the name of a single, monolithic markup language, but the name of a family of document types which collectively form a family of related markup languages. Since XHTML is a bit stricter than HTML the following should be kept in mind:

  1. XHTML elements must be properly nested
  2. XHTML element and attribute names must be in lowercase because XML is case sensitive
  3. XHTML elements must always be closed (trailing-slash sintax <br />)
  4. XHTML attribute values must always be quoted
  5. XHTML does not support attribute minimization, so value is required checkt="checked"
  6. XHTML follows a content model (ex. in <ul> elements only <li> elements can be included not <p>)
  7. The best place for style and script elements is in the external files (to evoid problems with rendering spetial charachters < or & or ]]> or --)
Have a look at the main points of Compatibility Guideline and recommended example with the best practices incorporated from W3C page.

important for XHTML

The root element of the document (html) must contain an xmlns declaration for the XHTML namespace

 < html xmlns="http://www.w3.org/1999/xhtml" xml:lang="en" lang="en" >

More about headers

W3C XHTML Media Types gives simple recommendtion about choosing media types:
In general, 'application/xhtml+xml' should be used for XHTML Family documents, and the use of 'text/html' should be limited to HTML-compatible XHTML Family documents intended for delivery to user agents that do not explicitly state in their HTTP Accept header that they accept 'application/xhtml+xml' [HTTP]. The media types 'application/xml' and 'text/xml' may also be used, but whenever appropriate, 'application/xhtml+xml' or 'text/html' should be used rather than those generic XML media types.
Therefore to maintain cross browsers and backwords compatibility the following declaration is a preferable one: 

<head>
<meta http-equiv="Content-Type" content="text/html; charset=utf-8" >
<title>Page title here</title>                                          
</head>
It is a good habit to declear charset before <title>, because in this case some special characters such as ♥ or © can be shown correctly in the page title.

Compatibility Guideline. Choosen poins from W3C Working Group Note (appendix A):

This appendix summarizes design guidelines for authors who wish their XHTML documents to render on both XHTML-aware and modern HTML user agents. The purpose of providing these guidelines is to supply a simple collection that, if followed, will give reasonable, predictable results in modern user agents. Document authors should treat these as best practices.
  1. DO use external scripts if your script uses < or & or ]]> or --. DO NOT embed a script in a document if it contains any of these characters. Rationale: While XML provides the CDATA structure to embed data such as this, that method will not work correctly should the document be delivered as media type text/html. The best way to insulate your scripts and stylesheets is to make them external. Both XML and HTML have mechanisms that can protect this content when it is embedded in a document. If you must embed the content, the following patterns can be used: Portably escaping embedded script contents: Portably escaping embedded style contents:
  2. DO ensure that attribute values are on a single line and only use single whitespace characters.
  3. DO NOT use line breaks and multiple consecutive white space characters within attribute values.
  4. DO use both lang and xml:lang attributes when specifying the language of an element in markup languages that support the use of both. Rationale: HTML 4 documents use the lang attribute to identify the language of an element. XML documents use the xml:lang attribute. CSS has a "lang" pseudo-selector that automatically uses the appropriate attribute depending on the media type. Therefore, specifying both attributes ensures that single CSS selectors will work in both modes.
  5. DO NOT use the name attribute to identify elements, even in languages that permit the use of name such as XHTML 1.0.
  6. DO ensure that the values used for the id attribute are limited to the pattern [A-Za-z][A-Za-z0-9:_.-]*.
  7. DO use explicit tbody elements within tables. Rationale: While the content model of the table element permits the tbody element to be skipped, in HTML 4 this element is implicit. HTML 4 user agents will silently add this element, thus potentially confusing scripts or style sheets.
  8. DO ensure that any CSS properties on the html element are also specified on the body element. Rationale: Some HTML user agents only recognize CSS properties on the body element.

Example from W3C Working Group Note (appendix B):

To have backword browsers compatibility and ensure that developed pages will adhere comming HTML5, XHTML5 standards the safe way is to use the following declaration and tags (in this example A.#N refers to W3C recommendations given in the Guidelines): 

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" 
                      "http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">
<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="en">

In this case, note that the lang attribute is not used on the html element. XHTML 1.1 does not support the lang attribute.

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" 
                      "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
<!-- use lang and xml:lang if your language supports it - A.7 -->
<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="en">
<head>
  

  <!-- Embed encoding when you can't set it from the server - A.9 -->
  <meta http-equiv="Content-Type"
        content="text/html; charset=utf-8" />
  
  <title>Example Portable XHTML</title>

  <!-- use base, not xml:base - A.20 -->
  <base href="http://www.w3.org/MarkUp" />

  <!-- use an external stylesheet or script - A.4, A.14 -->

  <link href="style/style.css" rel="stylesheet" type="text/css" />

  <!-- if you must embed, escape bad stuff - A.4, A.14 -->
  <style type="text/css">/*<![CDATA[*/
      /* Use lower case names in stylesheets - A.13 */
      .main { padding-left: 10px; }
      /* put properties on whole document - A.24 */
      html {background-color: #e4e5e9; }
      body {background-color: #e4e5e9; }
  /*]]>*/</style>

  <!-- Need full form of boolean attributes like defer - A.10 -->

  <script defer="defer" type="text/javascript">//<![CDATA[
      var SOME_VALUE="<this is a test>";
  //]]></script>

</head>

<body>

<!-- use the id attribute to identify elements - A.8 -->

<div id="main">

    <!-- ensure there are never linebreaks in attribute values - A.5 -->
    <h1 title="This section shows how to write portable content">
        Main Section
    </h1>
    <!-- IMG can never have content so use shorthand form - A.2 -->
    <img src="http://www.w3.org/Icons/w3c_main" alt="W3C logo" />  	

    <!-- An element with no content cannot use shorthand - A.3 -->

    <p></p>
    
    <!-- Apostrophes must be escaped using numeric - A.16 -->
    <p title='rule for &#39;ampersands&#39;'>

        <!-- Ampersands must be escaped - A.12 -->
        Some material &amp; some other material

    </p>

    <table>
        <!-- use explicit tbody element - A.19 -->
        <tbody>
            <tr>
                <td>Table Cell</td>

            </tr>
        </tbody>
    </table>

</div>

</body>
</html>

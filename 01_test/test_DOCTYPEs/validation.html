Validation

(fromsource) Validation is a process whereby documents are verified against the associated DTD, ensuring that the structure, use of elements, and use of attributes are consistent with the definitions in the Document Type Declaration (DTD).

  1. Validation is vital to have the site will be rendered consistently in different browsers.
  2. Validation is the first step towards SEO as valid markup is easier to read for search engines
  3. Faster loading as users' agents do not need to fix the pages
  4. Validation helps to avoid hacks so it provides forward-compatibility
  5. Validation helps maintain the site later on
  6. Validation saves time. Visual check in different browsers could not be enough as browsers do their best to represent the page with broken tags, but as more code added the best efforts could not be enough to fix all the problems. Even if the last bit of added code is correct, it still can look ugly because of the previous bit. So it is hard to identify the problem. All the browsers will reflect the valid design in the same way (or close enough), but they will show different results for invalid page. To save the time it is wise to validate the page instead of try to fix it with different tricks while it is still invalid and therefore no guaranty it will be represented correctly with the next bit of code added.
  7. Validation is a start for passing Web Content Accessibility Guidelines (WCAG) test.
  8. Validation icon on the page will prove that designer follows professional standards
